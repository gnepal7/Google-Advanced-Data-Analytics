# Tune a decision tree
  + Hyperparameters
    - Parameters that can be set before the model is trained.
  + Max depth
    - Defines how 'long' a decision tree can get.
  + Min samples leaf
    - Defines the minimum number of samples for a leaf node.

  + GridSearch
    - A tool to confirm that a model achieves its intended purpose
      by systematically checking every combination of Hyperparameters
      to identify which set produces the best results, based on 
      the selected metric.

# Hyperparameter tuning
  + Hyperparameter tuning
    - Models must learn patterns that generalize well to unseen data.
    - Overfitting occurs when models learn noise and perform poorly on new data.
    - Underfitting happens when models fail to capture important patterns in data.
    - Good models balance fitting training data without capturing randomness or noise.
    - Hyperparameters control model behavior and are tuned before training begins.
  + Hyperparameters for decision trees
    => max_depth
      - Max depth controls how many levels a decision tree can grow from root to leaf.
      - Very deep trees can overfit training data, while shallow trees may underfit.
      - Best models balance depth to generalize well without memorizing noise.
    => min_samples_split
      - min_samples_split sets minimum samples required for a node to split further.
      - Higher values make the tree stop growing earlier, producing simpler models.
      - Smaller thresholds allow deeper splitting, while larger ones may leave mixed nodes.
    => min_samples_leaf
      - min_samples_leaf sets the minimum samples required in each child node after a split.
      - It differs from min_samples_split by constraining child node size, not parent size.
      - Larger values prevent splits that create very small or uneven leaf nodes.
  + Finding the optimal set of hyperparameters
    => Grid search
      - Grid search trains models for every combination of specified hyperparameter values.
      - It evaluates combinations to find the best balance of fit and generalization.
      - Each hyperparameter combination produces a separate model for comparison.
      - The total models equal the product of all hyperparameter value options.
      - Some combinations increase regularization by limiting tree complexity.
      - Grid search can become computationally expensive with many parameters.
      - Efficient strategies include iterative narrowing or using broader initial ranges.

# Verify performance using validation
  + Model validation
    - The set of processes and activities intended to verify that
      models are performing as expected.
  + Cross validation
    - A process that uses different portions of the data to test and 
      train a model on different iterations.

# More about validation and cross-validation
  + Model validation
    - Test data should evaluate only the final model, not select the best model.
    - Choosing models using test data creates an overly optimistic performance estimate.
    - Model selection based on test results indirectly tunes the model.
    - Validation compares multiple models before choosing the best-performing candidate.
    - Proper validation provides a more objective assessment of model performance.
  + Validation sets
    - A validation set compares models while preserving the test set for final evaluation.
    - Separate validation works best with large datasets providing enough representative validation samples.
    - Retrain the chosen model using training and validation data before final testing.
  + Cross validation
    - Cross-validation splits training data into k folds for repeated training and validation.
    - Each fold serves once as validation, with average scores measuring overall performance.
    - Selected models are retrained using all training data after cross-validation completes.
    - Cross-validation reduces randomness and provides more reliable performance estimates.
    - It is especially useful for smaller datasets, though more folds increase training time.

# Tune and validate decision trees with Python
  import numpy as np
  import pandas as pd
  import matplotlib.pyplot as plt
  
  from sklearn.model_selection import train_test_split
  from sklearn.tree import DecisionTreeClassifier
  
  # This function displays the splits of the tree
  from sklearn.tree import plot_tree
  
  from sklearn.metrics import ConfusionMatrixDisplay, confusion_matrix
  from sklearn.metrics import recall_score, precision_score, f1_score, accuracy_score

  # Read in data
  file = 'Churn_Modelling.csv'
  df_original = pd.read_csv(file)
  df_original.head()

  # Check class balance
  df_original['Exited'].value_counts()

  # Calculate average balance of customers who churned
  avg_churned_bal = df_original[df_original['Exited']==1]['Balance'].mean()
  avg_churned_bal

  # Create a new df that drops RowNumber, CustomerId, Surname, and Gender cols
  churn_df = df_original.drop(['RowNumber', 'CustomerId', 'Surname', 'Gender'], 
                              axis=1)
  churn_df.head()

  + Feature transformation
  # Dummy encode categorical variables
  churn_df = pd.get_dummies(churn_df, drop_first=True)
  churn_df.head()

  + Split the data
  # Define the y (target) variable
  y = churn_df['Exited']
  
  # Define the X (predictor) variables
  X = churn_df.copy()
  X = X.drop('Exited', axis=1)
  
  # Split into train and test sets
  X_train, X_test, y_train, y_test = train_test_split(X, y, 
                                                      test_size=0.25, stratify=y, 
                                                      random_state=42)
  + Baseline model
  # Instantiate the model
  decision_tree = DecisionTreeClassifier(random_state=0)
  
  # Fit the model to training data
  decision_tree.fit(X_train, y_train)
  
  # Make predictions on test data
  dt_pred = decision_tree.predict(X_test)

  # Generate performance metrics
  print("Accuracy:", "%.3f" % accuracy_score(y_test, dt_pred))
  print("Precision:", "%.3f" % precision_score(y_test, dt_pred))
  print("Recall:", "%.3f" % recall_score(y_test, dt_pred))
  print("F1 Score:", "%.3f" % f1_score(y_test, dt_pred))

  + Analysis of baseline model
    def conf_matrix_plot(model, x_data, y_data):
      '''
      Accepts as argument model object, X data (test or validate), and y data (test or validate). 
      Returns a plot of confusion matrix for predictions on y data.
      ''' 
    
      model_pred = model.predict(x_data)
      cm = confusion_matrix(y_data, model_pred, labels=model.classes_)
      disp = ConfusionMatrixDisplay(confusion_matrix=cm,
                               display_labels=model.classes_)
    
      disp.plot(values_format='')  # `values_format=''` suppresses scientific notation
      plt.show()
  # Generate confusion matrix
  conf_matrix_plot(decision_tree, X_test, y_test)

  # Plot the tree
  plt.figure(figsize=(15,12))
  plot_tree(decision_tree, max_depth=2, fontsize=14, feature_names=X.columns, 
            class_names={0:'stayed', 1:'churned'}, filled=True);
  plt.show()

  + Tune and validate decision trees with Python
  # Import GridSearchCV
  from sklearn.model_selection import GridSearchCV

  # Instantiate the classifier
  tuned_decision_tree = DecisionTreeClassifier(random_state = 42)

  # Assign a dictionary of hyperparameters to search over
  tree_para = {'max_depth':[4,5,6,7,8,9,10,11,12,15,20,30,40,50],
             'min_samples_leaf': [2, 5, 10, 20, 50]}

  # Assign a set of scoring metrics to capture
  scoring = {'accuracy', 'precision', 'recall', 'f1'}

  %%time
  # Instantiate the GridSearch
  clf = GridSearchCV(tuned_decision_tree, 
                     tree_para, 
                     scoring = scoring, 
                     cv=5, 
                     refit="f1")
  
  # Fit the model
  clf.fit(X_train, y_train)

  # Examine the best model from GridSearch
  clf.best_estimator_

  print("Best Avg. Validation Score: ", "%.4f" % clf.best_score_)

  def make_results(model_name, model_object):
    '''
    Accepts as arguments a model name (your choice - string) and
    a fit GridSearchCV model object.
  
    Returns a pandas df with the F1, recall, precision, and accuracy scores
    for the model with the best mean F1 score across all validation folds.  
    '''

    # Get all the results from the CV and put them in a df
    cv_results = pd.DataFrame(model_object.cv_results_)

    # Isolate the row of the df with the max(mean f1 score)
    best_estimator_results = cv_results.iloc[cv_results['mean_test_f1'].idxmax(), :]

    # Extract accuracy, precision, recall, and f1 score from that row
    f1 = best_estimator_results.mean_test_f1
    recall = best_estimator_results.mean_test_recall
    precision = best_estimator_results.mean_test_precision
    accuracy = best_estimator_results.mean_test_accuracy
  
    # Create table of results
    table = pd.DataFrame()
    # Create table of results
    table = pd.DataFrame({'Model': [model_name],
                          'F1': [f1],
                          'Recall': [recall],
                          'Precision': [precision],
                          'Accuracy': [accuracy]
                         }
                        )
  
    return table

  # Call the function on our model
  result_table = make_results("Tuned Decision Tree", clf)

  # Save table as csv
  result_table.to_csv("Results.csv")

  # View the results
  result_table
