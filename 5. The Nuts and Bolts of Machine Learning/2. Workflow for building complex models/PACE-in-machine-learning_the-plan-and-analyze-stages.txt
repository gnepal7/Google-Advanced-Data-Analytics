# Introduction to feature engineering
  + Feature Engineering 
    - The process of using prectical, statistical, and data science
      knowledge to select, transform, or extract charaeteristics, 
      properties, and attributes from raw data.
  + Continuous variables
    - variables with values obtained by measurement; as a result, they
      can take on an infinite and uncountable set of values.
  + Categorical variables
    - Variables that contain a finite number of groups, categories, 
      or countable numerical values.

  + Three general categories of feature engineering
    - Selection
      > Select the features in the data that contribute the most to 
        predicting your response variable.
    - Transformation
      > Modifying existing features in a way that improves accuracy
        when training the model.
    - Extraction
      > Taking multiple features to create a new one that would improve
        the accuracy of the algorithm.

# Explore feature engineering
  + Feature Engineering
    - Model performance depends heavily on relevant, high-quality data 
      containing predictive features for target variables.
    - Irrelevant features reduce prediction quality, even when they 
      appear logically connected to outcomes.
    - Feature engineering selects, transforms, or extracts useful 
      characteristics from raw data for modeling effectiveness.
    - Effective feature engineering strengthens predictive signals, 
      improving machine learning model accuracy and overall performance.
  + Feature Selection
    => Three types of features are:-
      1. Predictive: Features that by themselves contain information 
        useful to predict the target                       
      2. Interactive: Features that are not useful by themselves to 
        predict the target variable, but become predictive in conjunction
        with other features
      3. Irrelevant: Features that don’t contain any useful information 
        to predict the target
  + Feature Transformation
    - Feature transformation modifies existing dataset features to 
      improve model training effectiveness.
    - It is typically performed in the Construct phase after analyzing 
      data and deciding transformations.
    => Log normalization
      > Some models perform poorly with continuous variables that 
        have skewed distributions.
      > Log normalization reduces skew by applying a logarithmic 
        transformation to features.
    => Scaling
      > Scaling is a feature transformation that adjusts feature value ranges 
        using normalization to prevent large-valued features from dominating 
        model predictions.
    => Normalization
      > Normalization (MinMaxScaler) rescales feature values so the minimum 
        becomes 0, maximum becomes 1, and all other values fall between 0 and 1.
    => Standardization
      > Standardization is a scaling technique that transforms features so they 
        have a mean of 0 and a standard deviation of 1 by subtracting the mean and 
        dividing by the standard deviation.
    => Encoding
      > Encoding is a feature transformation that converts categorical data 
        into numerical values so machine learning models can process and 
        analyze them mathematically.
  + Feature extraction
    - Feature extraction creates new variables from existing data 
      for better prediction.
    - It differs from transformation by generating new features, 
      not modifying existing ones.
    - Extracted features often combine or reinterpret raw data into 
      more useful forms.
    - Example: convert purchase date into days since last purchase for insight.

# Solve issues that come with imbalanced datasets
  + Class imbalance 
    - When a dataset has a predictor variable that contains more
      instances of one outcome than another.
  + Up-sampling
    - Fruitful for working with small dataset
  + Down-sampling
    - Useful for working with large set of data.

# More about imbalanced datasets
  + Imbalanced Datasets
    - Classification datasets require preprocessing like handling 
      missing values and proper data formatting.
    - Classes are unique categorical values, and their counts 
      form the class distribution.
    - Highly imbalanced class distributions can reduce model 
      performance for minority classes.
    - Class balancing adjusts data or training to reduce imbalance 
      and improve predictions.
  + Situations of Imbalance
    - Rare event classification involves target classes with 
      highly uneven frequencies.
    - Minority class occurs far less often than the majority 
      class in datasets.
    - Example: spam detection has far fewer spam emails 
      than non-spam emails.
    - Severe imbalance can cause models to ignore or poorly 
      predict minority class.

  + Downsampling
    - Downsampling reduces majority class size to balance 
      class distributions.
    - It is mainly used when datasets are large enough, 
      often tens of thousands of samples.
    - Random removal of majority class observations is a 
      common downsampling method.
    - Model performance must be checked to ensure data 
      reduction does not harm results.
  + Upsampling
    - Upsampling balances data by increasing minority 
      class observations instead of removing majority class data.
    - A simple method is duplicating existing minority class 
      samples multiple times.
    - Synthetic sampling creates new minority class data 
      points from existing observations.
    - Techniques like averaging or algorithms in Python generate 
      realistic synthetic samples.
